# Bistro Systems Teacher Guide

## How To Run the Soft Opening Debrief
Keep the debrief short and factual.

Use three prompts:
- What worked?
- Where did the system slow down?
- What will we practice next?

Redirect blame into system language:
- Role clarity
- Ticket flow
- Communication
- Timing
- Dish flow
- Sanitation
- Reset

End the debrief with one practice target students can see during the next activity.

## How To Reteach Stations
- Post station roles visibly.
- Give each station a setup job, service job, timing cue, ticket/task flow, and reset job.
- Have students say the role before starting.
- Rehearse station setup and reset before service pressure begins.
- Inspect station readiness before moving to mock service.

## How To Coach Role Clarity
Ask:
- "What is your role right now?"
- "What does your station own?"
- "Where does that ticket or task go next?"
- "Who needs to hear this?"
- "What can you reset while you wait?"

## How To Observe Ticket Flow
Watch for:
- Readable tickets
- Correct station handoff
- Clear call-backs
- Visible tickets
- Expo check before handoff
- Ticket/order closed or reset

## How To Run Mock Service 2.0
1. Start with one simple ticket path.
2. Confirm station roles before releasing students.
3. Run slowly enough to pause and correct.
4. Use one planned reset pause.
5. Practice one recovery moment: late ticket, unclear call-back, dish backup, or crowded expo.
6. Add a second round only if the first round is stable.
7. Debrief one improvement and one system to keep practicing.

## Common Student Mistakes
- Debriefing with blame
- Forgetting roles once service starts
- Not calling back ticket information
- Crowding expo or dish
- Waiting for the teacher to fix every problem
- Treating dish and reset as someone else's job
- Letting sanitation fade during service pressure

## Service Coaching Prompts
- "Call that back."
- "Keep the ticket visible."
- "What is the next step?"
- "What needs to be ready first?"
- "Who needs help before they fall behind?"
- "What can be reset right now?"
- "Would this station be useful to the next group?"
- "Fix the system, not the person."

## Cleanup and Reset Timing
- Build in one planned reset pause during mock service.
- Start dish flow before the final minutes.
- Check labels and storage before counters are wiped.
- Inspect stations before debrief.
- Require incomplete stations to fix reset immediately.

## Debrief Questions
- What improved from the soft opening?
- Which station was clearer today?
- Where did ticket flow slow down?
- What phrase helped communication?
- Was dish flow active during service?
- What reset job protected the next group?
