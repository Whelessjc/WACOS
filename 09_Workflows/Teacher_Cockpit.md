# Teacher Cockpit

## Purpose

The Teacher Cockpit is the complete instructor operating model for West Ashley Culinary. It shows where durable information belongs, which tool handles each kind of work, and how a working idea becomes an approved repository update.

## Source Hierarchy

When sources conflict, use this order:

1. Current instructor-confirmed decisions.
2. The [West Ashley Culinary Core Manual](../00_Core/West_Ashley_Culinary_Core_Manual.md).
3. Current WACOS operating documents.
4. Current subordinate documents and SOPs.
5. Archived material only as historical evidence.

GitHub is the permanent documentation record for WACOS. It does not replace instructor authority. Current instructor-confirmed decisions govern, and approved decisions are recorded in the repository so future work can use them reliably.

## Operating Loop

```text
WACOS GitHub Repository
          ↓
    ChatGPT Project
          ↓
     Working Chats
          ↓
  Generated Documents
          ↓
       Git Commit
          ↓
   Repository Update
```

The repository supplies current program context. ChatGPT supports working thought and document creation. The instructor reviews the result. Codex integrates approved permanent changes, validates them, commits them, and synchronizes GitHub.

## What Each System Does

### WACOS GitHub Repository

- Holds the permanent current record.
- Separates governing documents, operating materials, and historical sources.
- Preserves decision history and traceability.
- Supplies the authoritative context for source-bound AI work.

### ChatGPT Project

- Serves as the working environment for program planning.
- Uses Project Knowledge and Project Instructions to keep work aligned.
- Uses [`WACOS_ChatGPT_Quick_Reference.md`](../05_AI/WACOS_ChatGPT_Quick_Reference.md) for fast answers to common program questions.
- Uses the connected GitHub repository for detailed procedures, citations, recent changes, and questions not covered by the quick reference.
- Supports drafting, brainstorming, lesson development, document generation, and reflection.
- Produces drafts for instructor review, not automatic program decisions.

### Working Chats

- Handle one bounded problem, week, lesson sequence, Bistro need, or document at a time.
- May contain temporary reasoning, alternatives, and unfinished drafts.
- Are not the permanent source of truth.

### Memory

- Stores only stable preferences that remain useful across work.
- Should not hold changing calendars, student information, grades, temporary plans, or unapproved policy.
- Does not replace Project Knowledge or the repository.

### Codex

Codex handles:

- Repository edits and correct file placement.
- Cross-document consistency.
- Link, format, and validation checks.
- Decision, source, navigation, and changelog updates when required.
- Git commits and pushes after approval.

Permanent changes follow the [WACOS Update Workflow](../00_Core/WACOS_Update_Workflow.md).

### ChatGPT

ChatGPT handles:

- Planning and brainstorming.
- Lesson and lab development.
- Drafting and revision.
- Creating teacher and student document drafts.
- Organizing reflection and identifying possible next steps.

Student-facing materials always require instructor review. Safety, allergens, grading, discipline, district requirements, and program policy remain instructor-controlled.

## District Systems Remain External

District and business systems remain outside WACOS unless an approved connection is available. Examples include:

- Google Docs and Google Drive.
- Outlook or Gmail.
- Calendar.
- Canvas.
- Square.
- Bookkeeper forms.
- Printing and copying systems.

Use these systems for their assigned operational purpose. Record only durable program documentation in WACOS. Do not place student records, credentials, private financial information, or district-restricted data in the repository or an unapproved AI system.

## Weekly Control Pattern

- **Monday-Wednesday:** protect instruction, practice, formative checks, grading, and next-week planning.
- **Thursday:** execute Bistro Prep and leave the room turnkey for Friday.
- **Friday:** execute Bistro Service, close the kitchen and register, complete required external paperwork, and reflect.
- **Weekend:** review pacing, inventory, curriculum, and possible WACOS changes.

Use the [Weekly Teacher Workflow](Weekly_Teacher_Workflow.md) for the day-by-day sequence and the [Instructor Recurring Checklists](Instructor_Recurring_Checklists.md) for repeatable checks.

## Decision Gate

Before making a permanent change, ask:

1. Is this a temporary weekly adjustment or a durable WACOS change?
2. Which current document governs it?
3. Does it change policy, curriculum, Bistro operations, assessment, safety, or district procedure?
4. Does it need instructor confirmation, a Decision Register entry, or a source record?
5. Has the final material been reviewed before student use or publication?
