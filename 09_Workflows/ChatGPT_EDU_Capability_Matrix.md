# ChatGPT EDU Capability Matrix

Current environment snapshot: July 22, 2026

## Purpose

This matrix records the instructor's current ChatGPT EDU working environment. It is operational guidance, not a promise of district access. District permissions, product features, and connectors may change.

## Available

| Capability | Current use | Boundary |
|---|---|---|
| Projects | Organize WACOS work in a persistent working environment. | A Project is not the permanent source of truth. |
| Project Knowledge | Supply current WACOS references for source-bound work. | Keep sources current and exclude restricted data. |
| Project Instructions | Preserve stable WACOS working rules and terminology. | Instructions do not override the Core Manual or instructor decisions. |
| Memory | Retain stable preferences. | Do not store students, grades, temporary plans, credentials, or changing policy. |
| GitHub | Read or synchronize the permanent WACOS record through approved workflows. | Approved repository content remains the durable record. |
| Python | Analyze information and support file generation. | Instructor reviews outputs and assumptions. |
| Word generation | Draft teacher, student, and administrative documents. | Student-facing and official documents require instructor review. |
| Excel generation | Draft trackers, tables, inventories, and planning workbooks. | Does not replace the district gradebook or financial system. |
| PowerPoint generation | Draft lesson and presentation materials. | Verify content, links, and classroom fit before use. |
| PDF generation | Produce fixed-layout delivery copies. | Verify the source and final rendering. |
| Image generation | Create draft instructional or program visuals. | Review accuracy, appropriateness, rights, and student-facing use. |
| Voice | Support spoken brainstorming and drafting. | Confirm important decisions in writing before permanent use. |
| Automations | Schedule supported reminders or recurring AI tasks. | Do not treat automation as confirmation that external work is complete. |

## Blocked in the Current District Environment

| Connector | Current status | Operating approach |
|---|---|---|
| Google Drive | Blocked | Use district-approved Drive and Google Docs manually as external systems. |
| Gmail | Blocked | Use the approved email system directly. |
| Google Calendar | Blocked | Review and update the approved calendar directly. |
| Canvas connector | Blocked | Use Canvas directly for course publishing and grade-related work. |

District permissions may change. Retest a blocked connector only when a new district-approved option becomes available; do not repeatedly test unchanged permissions.

## External Systems

Google Docs, email, calendars, Canvas, Square, bookkeeper forms, printing, and other district systems remain external even when ChatGPT helps draft content for them. The instructor completes final review, transfer, publication, and any required official action.

## Review Rules

- Use the [Teacher Cockpit](Teacher_Cockpit.md) for the complete tool workflow.
- Follow the [WACOS AI Reference](../00_Core/WACOS_AI_Reference.md) and [AI Workflows](../05_AI/11_AI_Workflows.md).
- Do not upload restricted student, personnel, financial, credential, or district data to an unapproved system.
- Do not let AI make final grading, safety, allergen, discipline, legal, or policy decisions.
- Review all student-facing materials before use.
- Date and revise this matrix when the actual environment changes.
