# WACOS

West Ashley Culinary Operating System

## What This Repository Is

WACOS is the authoritative documentation home for the West Ashley High School Culinary Arts Program.

It organizes the program's governing documents, curriculum, Bistro operations, assessment systems, recipes, AI guidance, templates, administration, equipment and safety materials, instructor workflows, and historical archive.

The repository is designed for practical use during the school year. A teacher should be able to find the current source of truth quickly, create new documents in the right place, and avoid mixing current practice with retired or historical material.

## Start Here

Begin in `00_Core/`.

If you are returning after time away or want to add, change, or remove something, begin with [`00_Core/WACOS_Update_Workflow.md`](00_Core/WACOS_Update_Workflow.md). It provides the fast update lanes, copy-ready request language, approval shortcuts, and the return-status prompt.

The Core Manual governs the rest of the repository. Current documents should align with:

- `00_Core/WACOS_Update_Workflow.md`
- `00_Core/West_Ashley_Culinary_Core_Manual.md`
- `00_Core/WACOS_Playbook.md`
- `00_Core/WACOS_Alignment_Checklist.md`
- `00_Core/Core_Manual_Traceability.md`

If a document conflicts with the Core Manual, revise the document or flag the issue for instructor review.

## Source Authority

Current authority follows this order:

1. Current instructor-confirmed decisions.
2. The Core Manual.
3. Current WACOS operating documents.
4. Traceability and source records.
5. Archive materials for historical context only.

Archived material is historical unless explicitly reapproved. Old ProStart-centered documents and older grading structures should not be treated as current program policy.

## Maintenance Rule

When adding or revising a document:

1. Choose the smallest lane in the WACOS Update Workflow.
2. Check the Core Manual when the change affects program direction.
3. Use the WACOS Alignment Checklist.
4. Put the file in the correct folder.
5. Mark historical or uncertain material clearly.
6. Update traceability, changelog, or decision records when the change affects program direction.
